Оптимизации, выполненные в проекте:

1. Логирование:
- Внедрена ротация и сжатие лог-файлов с использованием библиотеки lumberjack, что предотвращает бесконтрольный рост логов и снижает нагрузку на диск.
- Добавлен параметр debug для управления уровнем логирования (debug / warn), позволяющий уменьшать объем логов в продакшене и снижать накладные расходы.
- Исключено избыточное логирование — ведется логирование только ошибок в горячих участках кода на уровне HTTP-хендлеров.

2. HTTP сервер:
- Настроены таймауты ReadTimeout, WriteTimeout и IdleTimeout (10 секунд для чтения/записи, 120 секунд для ожидания), что улучшает устойчивость и производительность при обработке запросов.
- Обновлена проверка ошибок при запуске сервера с учетом корректного завершения (Http.ErrServerClosed).

3. HTTP Хендлеры:
- Оптимизирован процесс работы с JSON: добавлен комментарий к возможности использования методов для эффективного биндинга данных (ShouldBindBodyWith).
- Минимизирована нагрузка на логирование в обработчиках, логируются только ошибки.

4. Основной модуль (cmd/main.go):
- Инициализация логгера с параметром debug из конфигурационного файла.
- Улучшено управление сервером — корректная обработка сигналов на завершение и корректный shutdown с таймаутом.
- Добавлена импортировка пакета net/http, устранена критическая ошибка компиляции.

Данные оптимизации направлены на повышение производительности, уменьшение нагрузки на ресурсы при работе сервиса, повышение надежности и управляемости логирования.

С Даннымим оптимизациями было проведено нагрузочное тестирование в Postman с около 6000 итерациями.
